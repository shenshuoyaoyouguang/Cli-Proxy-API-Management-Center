# 使用统计数据可视化优化计划

## 一、现有可视化系统分析

### 1.1 已实现的统计指标

| 指标类别 | 具体指标 | 展示方式 | 数据来源 |
|---------|---------|---------|---------|
| **请求统计** | 总请求数、成功/失败数 | StatCards + Sparkline | `usage.total_requests`, `success_count`, `failure_count` |
| **Token统计** | 总Token数、缓存Token、推理Token | StatCards + Sparkline | `tokens.input_tokens`, `output_tokens`, `cached_tokens`, `reasoning_tokens` |
| **速率指标** | RPM、TPM（30分钟窗口） | StatCards + Sparkline | 实时计算 `calculateRecentPerMinuteRates()` |
| **成本指标** | 总花费 | StatCards + Sparkline | 基于模型价格计算 `calculateCost()` |
| **服务健康** | 7天成功率热力图 | ServiceHealthCard (7×96网格) | `calculateServiceHealthData()` |
| **API详情** | 各端点请求数、成功率、Token、成本 | ApiDetailsCard（可展开表格） | `getApiStats()` |
| **模型统计** | 各模型请求数、Token、成本 | ModelStatsCard（排序表格） | `getModelStats()` |

```

│   │  当前利用率: 45%  │  配额: 100 RPM                   │   │
│   └─────────────────────────────────────────────────────┘   │
│                                                             │
│   迷你趋势图 (最近60分钟)                                   │
│   ┌─────────────────────────────────────────────────────┐   │
│   │      ╱╲                                              │   │
│   │     ╱  ╲    ╱╲                                       │   │
│   │    ╱    ╲  ╱  ╲     ╱╲                              │   │
│   │───╱──────╲╱────╲───╱──╲────────────────────────────│   │
│   └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

**实现要点**:

1. **多值展示**:
   - 当前值（最近1分钟）
   - 峰值（窗口内最大值及时间）
   - 平均值（窗口平均值）

2. **配额利用率条**:
   - 整合 `useQuotaStore` 中的配额数据
   - 可视化展示当前使用占配额的百分比
   - 颜色编码：绿色(<60%) → 黄色(60-80%) → 红色(>80%)

3. **趋势迷你图增强**:
   - 显示峰值标记点
   - 添加参考线（配额线）

**数据结构设计**:

```typescript
interface EnhancedRateStats {
  current: number;           // 当前值（最近1分钟）
  peak: number;              // 峰值
  peakTime: string;          // 峰值时间
  average: number;           // 平均值
  windowMinutes: number;     // 统计窗口
  trend: number[];           // 趋势数据（每分钟一个点）
  quota?: {                  // 配额信息（可选）
    limit: number;           // 配额上限
    utilization: number;     // 利用率 (0-1)
  };
  changePercent: number;     // 相比前一分钟的变化百分比
}
```

## 七、数字展示优化方案

### 7.1 当前问题分析

**现状代码位置**: [usage.ts:392-442](file:///d:/xiaoxiao/2026.3.3/Cli-Proxy-API-Management-Center-main/src/utils/usage.ts#L392-L442)

```typescript
// 当前 formatCompactNumber 实现
export function formatCompactNumber(value: number): string {
  if (abs >= 1_000_000) {
    return `${(num / 1_000_000).toFixed(1)}M`;  // 仅支持 M
  }
  if (abs >= 1_000) {
    return `${(num / 1_000).toFixed(1)}K`;       // 仅支持 K
  }
  return abs >= 1 ? num.toFixed(0) : num.toFixed(2);
}
```

**问题清单**:
1. ❌ 单位固定（K/M），不支持国际化单位（万/亿）
2. ❌ 不支持用户自定义单位偏好
3. ❌ 缺乏上下文感知（如Token数与金额应不同展示）
4. ❌ 小数位数固定，不够灵活
5. ❌ 缺乏完整数字与紧凑格式的切换选项

### 7.2 优化方案

// 中文单位格式化
export function formatChineseUnit(value: number, decimals?: number): string {
  const abs = Math.abs(value);
  if (abs >= 1e12) return `${(value / 1e12).toFixed(decimals ?? 2)}万亿`;
  if (abs >= 1e8) return `${(value / 1e8).toFixed(decimals ?? 2)}亿`;
  if (abs >= 1e4) return `${(value / 1e4).toFixed(decimals ?? 2)}万`;
  return value.toLocaleString();
}

// 英文单位格式化
export function formatEnglishUnit(value: number, decimals?: number): string {
  const abs = Math.abs(value);
  if (abs >= 1e12) return `${(value / 1e12).toFixed(decimals ?? 1)}T`;
  if (abs >= 1e9) return `${(value / 1e9).toFixed(decimals ?? 1)}B`;
  if (abs >= 1e6) return `${(value / 1e6).toFixed(decimals ?? 1)}M`;
  if (abs >= 1e3) return `${(value / 1e3).toFixed(decimals ?? 1)}K`;
  return value.toLocaleString();
}

// Token专用格式化（更精确）
export function formatTokenNumber(value: number, options?: NumberFormatOptions): string {
  // Token数通常较大，使用更紧凑的格式
  // 但保留足够精度用于成本计算
}

// 成本专用格式化
export function formatCostNumber(value: number): string {
  // 金额总是显示2位小数
  // 小额显示更多精度（如 $0.00123）
  const abs = Math.abs(value);
  if (abs < 0.01 && abs > 0) {
    return `$${value.toFixed(6)}`;
  }
  return `$${value.toFixed(2)}`;
}
